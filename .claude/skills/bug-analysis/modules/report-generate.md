# 报告生成模块

## 职责
生成分析报告，推送到飞书。

## 输入
- 分析结果
- 案例匹配结果
- 知识库引用

## 输出
- Markdown 报告
- 飞书云文档 URL

## 报告结构

> **核心原则：结论先行**。报告必须先给结论、再给分析。第一个内容章节是「结论先行」，用 3~5 句话讲清问题定性、根本原因、关键证据、影响范围、首要建议；之后的章节才展开详细推导与日志依据。

### 0. 结论先行（硬规则，置于报告最前）
- 问题定性（问题类型 + 软件/射频/配置/环境归属）
- 根本原因（一句话）
- 关键证据（1~2 条最有力的日志/数据）
- 影响范围
- 首要建议（最直接有效的一条）

### 1. 日志完整性（硬规则）
- 列出 4 类日志的存在状态（main log / kernel log / tcpdump / 空口 log）
- 缺失日志必须标注 **未提供**
- 缺失日志对分析结论的影响说明

### 2. 项目匹配
- 匹配项目
- 匹配案例
- 分析依据

### 3. 总体统计
- 总尝试次数
- 成功/失败次数
- 成功率

### 4. 每次尝试详细分析
- 时间
- 阶段
- 耗时
- 结果

### 5. 失败原因分类
- 模式名称
- 次数
- 特征

### 6. 关键日志
- 失败日志
- 成功日志（对比）

### 7. 问题原因
- 根本原因
- 具体分析

### 8. 流程总结
- 成功路径
- 失败路径

### 9. 建议措施
- 排查建议
- 优化建议

### 10. 案例素材
- 可追加到案例库的格式

## 执行步骤

### 步骤 1: 加载报告模板
- 读取 `templates/report-template.md`

### 步骤 2: 填充数据
- 替换模板中的占位符
- 插入分析结果
- 插入案例匹配结果
- **插入曲线图** — 在「链路质量曲线图」章节使用 `![描述](文件名)` 嵌入 PNG 图片
  - `{ISSUE_KEY}_link_quality.png` — 链路质量四象限图
  - `{ISSUE_KEY}_kernel_metrics.png` — Kernel Metrics 图
  - `{ISSUE_KEY}_timeline.png` — 连接状态时间轴图（从 main_log 生成）
  - 图片文件必须与报告文件保存在同一目录

### 步骤 3: 保存报告
- 保存到 `<ISSUE_KEY>/<ISSUE_KEY>-analysis.md`

### 步骤 4: 推送到飞书
- 调用飞书 API
- 返回云文档 URL

### 步骤 5: 更新报告
- 在报告末尾添加飞书链接

## 飞书推送

### 推送命令
```bash
python scripts/feishu_import_docx.py --file <报告路径> --title "<标题>"
```

### 推送内容
- 完整报告原文
- 不生成精简版

## 硬规则
0. **结论先行** — 报告第一个内容章节必须是「结论先行」，先给结论（问题定性、根本原因、关键证据、影响、首要建议），再展开详细分析。严禁把结论埋在报告中后部。
1. **案例匹配结果必须出现在报告中**
2. **报告必须保存到本地**
3. **飞书推送必须执行**
4. **日志完整性必须紧随「结论先行」之后呈现** — 按 checklist 格式列出 4 类日志状态，缺失日志注明对分析的影响
5. **曲线图必须插入报告** — 有 kernel log 时，生成的 `{ISSUE_KEY}_link_quality.png` 和 `{ISSUE_KEY}_kernel_metrics.png` 必须以 `![]()` 语法嵌入报告正文，图片与报告同目录
6. **连接状态时间轴必须插入报告** — 有 main_log 时，生成的 `{ISSUE_KEY}_timeline.png` 必须以 `![]()` 语法嵌入报告正文
