# CASE-003: 手机向PC分享图片/文档提示连接失败（P2P连接成功后DHCP失败）

## 基本信息
- **案例ID**: CASE-003
- **分类**: dhcp-failure
- **来源**: OS162-40436
- **创建时间**: 2026-06-01
- **匹配次数**: 0

## 现象描述
- TECNO CN5c (MTK平台) 向PC分享图片/文档时，提示连接失败
- WiFi层面P2P连接成功建立，4次握手完成
- 存在两种DHCP失败模式：
  1. Kernel层DHCP成功但Android IpClient框架获取APF能力失败（ENOSYS）
  2. Kernel层DHCP REQUEST阶段失败，GO端未回复ACK
- 最终IP地址无法配置到p2p0接口，应用层获取到remoteIp=null, localIp=null
- 超时断开，错误码：WIFI_P2P_TIMEOUT

## 根因结论
**模式1**：P2P接口(p2p0)不支持APF（Android Packet Filter），IpClient获取APF能力时返回ENOSYS错误，导致DhcpClient初始化失败，无法使用Kernel层已获取的DHCP结果。

**模式2**：GO端（PC）DHCP服务异常，收到DISCOVER并回复OFFER，但未处理REQUEST或ACK丢失，导致Kernel层DHCP REQUEST重传4次后失败。

## 排查步骤
1. 通过 `notifyConnectFail reason=WIFI_P2P_TIMEOUT` 定位失败事件
2. 检查WiFi连接日志，发现4次握手成功完成
3. 检查kernel_log，发现两种DHCP失败模式：
   - 模式1：DHCP四步握手成功，但Android框架层IpClient报ENOSYS错误
   - 模式2：DHCP REQUEST重传4次，未收到ACK
4. 检查netd日志，发现连接无interfaceSetCfg，IP地址从未配置到p2p0接口
5. 对比之前成功的连接，确认问题出在DHCP流程
