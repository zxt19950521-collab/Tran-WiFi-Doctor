# CASE-002: 双端 2.4G 网络下 Android 碰传 Android 偶现发送失败

## 基本信息
- **案例ID**: CASE-002
- **分类**: p2p-connection
- **来源**: OS162-34615
- **创建时间**: 2026-05-29
- **匹配次数**: 0

## 现象描述
- 双端连接 2.4G WiFi 时，Android 碰传 Android 偶现发送失败
- 机型：双 KN4N (TECNO SPARK 50)，MTK 平台
- 复现率：2/20 = 10%
- GO 端 21/21 全部成功开组，GC 端 19/21 成功
- 失败发生在 P2P Group Formation 阶段，未进入数据面

## 根因结论
**MTK driver 双端 RX 路径在 SCC 同信道 + AP 密集环境下双向失能**

具体表现为两种形态：
1. **形态 A（本次复现）**：GC 端 driver 在目标 ch1 上未上报 GO 的 Probe Response → 3 次定向扫描后 P2P-GROUP-FORMATION-FAILURE
2. **形态 B（历史复现）**：扫描到 + 发 Auth/AssocReq，但 GO 重传 Assoc Resp，GC RX 异常

## 排查步骤
1. 通过 `notifyConnectFail reason=WIFI_P2P_TIMEOUT` 定位失败事件
2. 检查 kernel log 中 `ScanReqV2` 和 `ScanDone` 统计，发现 ch1 MdrdyCnt 极低（0/11/5）
3. 使用空口 sniffer pcap 分析，发现 GC ProbeReq 正常发出但 GO 未响应
4. 发现 GO 端 driver ProbeReq 响应延迟 ≥ 2.8 秒（GC 已 timeout 之后才回）
5. 确认根因为 MTK driver 在 SCC 同信道 + AP 密集环境下 RX 路径被 coex 调度抢断
