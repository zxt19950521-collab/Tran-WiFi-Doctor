# CASE-006: 蜂窝热点长时间开启导致温升，伴随TetherOffload/BPF网络栈报错

## 基本信息
- **案例ID**: CASE-006
- **分类**: performance
- **来源**: LK7OS163-2134
- **创建时间**: 2026-06-08
- **匹配次数**: 0

## 现象描述
- LK7（TECNO XYZ）印度尼西亚粉丝反馈：开启热点时手机过热
- 蜂窝热点（MOBILE，上游 ccmni2）灭屏运行约 70 分钟
- 电池温度 38.5→44.6°C（+6.1°C），CPU 温度 47.3°C
- 平均放电电流 229mA，耗电几乎全部由 modem 热点导致
- 信号约 2 格（XL 运营商），属中等偏差
- 热点网络栈持续报错：TetherOffload readFD fail、BpfCoordinator conntrack 更新失败

## 根因结论
**主因**：蜂窝热点长时间灭屏运行，modem 持续高功耗发射/接收，在中等信号环境下导致温升。

**次因（待 modem 团队确认）**：TetherOffload 硬件卸载路径持续失败（readFD err=-4，1300+ 次）及 BpfCoordinator conntrack 更新失败（108 次），可能导致数据包处理回退到 CPU 软路径，增加额外功耗和发热。

## 排查步骤
1. 确认热点类型：`mActiveNetworkType : MOBILE`，流量接口 `ccmni2`（蜂窝热点，非 WiFi 回传）
2. 检查温升时段是否与热点开启时长重叠
3. 搜索 `TetherOffload` 报错：`readFD fail err=-4`、`getDataUsage fail errno: 2`
4. 搜索 `BpfCoordinator`：`Failed to update conntrack entry`、`onAlertReached`
5. 检查 `tetherGetStats` 中 ccmni2 流量是否持续增长
6. 排除高 CPU 应用占用（本案例已排除）
7. 检查 `board-temperature-changed` 温控策略是否触发
8. 对比 radio_log 信号强度（RSRP/level）
